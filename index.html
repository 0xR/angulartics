Angulartics

Web analytics for AngularJS applications

Getting started

Installation

While the only currently supported provider is Google Analytics, which will be shown in the following examples, Angulartics is vendor-agnostic. I expect to release additional providers in the short-term.

To start using Angulartics for Google Analytics:

1. Embed your GA code into your HTML main page (not the templates!), making sure you delete the page view tracking line.

2. In your AngularJS app definition, add Angulartics and the Angulartics GA plugin to your dependencies.

index.html

    <html ng-app="myApp">
        ...
        <script type="text/javascript">
            var _gaq = _gaq || [];
            _gaq.push(['_setAccount', 'YOUR_UA_CODE']);
            _gaq.push(['_trackPageview']);
            ...
        </script>

Virtual page tracking

Angulartics provides automatic virtual page tracking out of the box, meaning the entire user navigation across the different routes of your application is tracked. It can always be turned off with autoTracking(false).

 

app.js

    angular.module('myApp', ['angulartics', 'angulartics.ga'])
    .config(function($analyticsProvider) {
      // $analyticsProvider.autoTracking(false);

Using the API

In order to track page views and events from within your application logic, inject $analytics and invoke either the pageTrack() or eventTrack() methods.

 

JavaScript

  module.controller('SampleCtrl', function($analytics) {
    // virtual page tracking
    $analytics.pageTrack('/my/url');

    // if using GA, params map to Category, Action, Label, Value
    $analytics.eventTrack(
      'actionType', 'action', 'identifier', 'value');

Declaratively in the HTML

Elements with attribute analytics-track will trigger Event Tracking. You also need to specify:

  1. the trigger event (default 'click');
  2. the action to be tracked;
  3. the action type (default 'Event').

While the action is mandatory, if it is omitted then Angulartics will try to infer it from the HTML element. For example, for a <button> it will use the inner text of the element. In the case of an <input> button, the value attribute will be used.

 

HTML

    <a href="#" ng-click="zoom()" 
        analytics-track="click" analytics-action="Zoom-in"
        analytics-action-type="Mapping">Zoom</a>

    <!-- event="click", action="Do Something" -->
    <button ng-click="doSomething()" 
        analytics-track 
        analytics-action-type="Commands">Do Something</button>

    <!-- event="click", category="Event", action="Do Something" -->
    <input type="submit" value="Step 3" analytics-track>

Enable tracking for multiple elements

You can declare tracking parameters for a group of elements by enclosing them in directive comments, which act as variables. This makes for less redundant code.

 

index.html

    <!-- directive: analytics-action-type Video -->
    <button ng-click="play()"></button>
    <button ng-click="stop()"></button>
    <button ng-click="mute()"></button>
    <!-- directive: analytics-action-type -->

Plugins

Google Analytics

The GA plugin implements Page Tracking and Event Tracking. To use it, just download angulartics-ga.js and add 'angulartics.ga' to your app dependencies.

Contribute!

You have an analytics vendor of choice and there's no Angulartics plugin for it? Please feel free to write your implementation. Contributions are most welcomed!
