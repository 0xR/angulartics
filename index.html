Angulartics

Web analytics for AngularJS applications

Follow @angulartics Tweet I use it

Vendor-agnostic & extensible

Angulartics has an abstract, extensible design; which means you can write your own plugin for your vendor of choice and contribute it to the community :)

Ready to use

Two of the most popular vendors are supported out of the box: Google Analytics and Kissmetrics.

1-line setup

Just add Angulartics to your app dependencies and you'll get automatic pageview tracking for your application routes.


Getting started

Installation

  1. Embed your vendor code into your HTML main page, making sure you delete any automatic tracking line.
index.html
  <script>
      var _gaq = _gaq || [];
      _gaq.push(['_setAccount', 'YOUR_UA_CODE']);
      _gaq.push(['_trackPageview']);
      ...
  1. In your AngularJS app dependencies, inject Angulartics and the Angulartics vendor plugin of your choice.
app.js
  angular.module('myApp', ['angulartics', 'angulartics.ga'])

Virtual pageviews

Angulartics provides automatic virtual pageview tracking out of the box, meaning the entire user navigation across the different routes of your application is tracked. It can always be turned off with virtualPageviews(false).

app.js
  .config(function($analyticsProvider) {
    // if you want to turn off automatic tracking, just set:
    $analyticsProvider.virtualPageviews(false);

Declarative event tracking

The simplest way to do event tracking is by adding the attributes analytics-on and analytics-event to the element.

Think of it as: "on click, track event name".

HTML
  <a href="file.pdf" 
    analytics-on="click" 
    analytics-event="Download">Download</a>

Inferred values

analytics-on needs to be present, because it is the attribute that enables event tracking for the element. However, Angulartics will try to infer its value if omitted.

For example, for <button> and <a> elements, it will be assumed that you want to track whenever they are clicked.

Same happens if analytics-event is not present or empty: following the example above, the event name will be obtained from the inner text of the element.

 

HTML
  <a href="file.pdf" 
    analytics-on="click" analytics-event="Download">Download</a>

  <!-- same as above -->
  <a href="file.pdf" analytics-on>Download</a>

  <!-- same as adding analytics-event="Step 3" -->
  <input type="submit" value="Step 3" analytics-on>

Additional parameters

Each vendor has its own set of tracking properties. For example, Google Analytics requires you to specify a category, and also lets you add a label and value. You can set these properties by adding attributes prefixed with analytics-.

 

HTML
  <button analytics-on
    analytics-event="Play"
    analytics-category="Videos"
    analytics-label="Gone with the Wind">Play<button>

Using the API

In order to track pageviews and events from within your application logic, inject $analytics and invoke either the pageTrack() or eventTrack() methods.

JavaScript
  module.controller('SampleCtrl', function($analytics) {
    $analytics.pageTrack('/my/url');
    $analytics.eventTrack('eventName');
    $analytics.eventTrack('eventName', { 
      category: 'category', label: 'label'
    });

Plugins

The GA plugin implements Page Tracking and Event Tracking. To use it, just add angulartics-ga.js and inject angulartics.ga into your app dependencies.


The KM plugin implements Event Tracking which is used both for virtual pageviews and events. You need to add angulartics-km.js and inject angulartics.km into your app dependencies.

Contribute!

You have an analytics vendor of choice and there's no Angulartics plugin for it? Please feel free to write your implementation. Contributions are most welcomed!
